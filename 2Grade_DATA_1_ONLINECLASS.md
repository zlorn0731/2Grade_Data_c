---

# ⏱ 알고리즘 성능 분석

알고리즘을 평가할 때 중요한 것은 **정확한 실행시간이 아니라 증가 속도**입니다.

알고리즘 성능은 크게 두 가지로 분석합니다.

| 구분 | 설명 |
|-----|-----|
| 시간 복잡도 (Time Complexity) | 알고리즘이 수행되는 연산 횟수 |
| 공간 복잡도 (Space Complexity) | 알고리즘이 사용하는 메모리 크기 |

일반적으로 **시간 복잡도 분석이 가장 중요합니다.**

---

# 📈 시간 복잡도 (Time Complexity)

시간 복잡도는 **입력 크기 n에 따라 실행 시간이 어떻게 증가하는지**를 나타냅니다.


T(n)


여기서


n = 입력 크기


---

# 📊 빅오 표기법 (Big-O Notation)

빅오 표기법은 **알고리즘 실행시간 증가율의 상한(Upper Bound)** 을 나타내는 방법입니다.

### 정의

두 함수 f(n), g(n)이 있을 때


|f(n)| ≤ c|g(n)|


을 만족하는 **상수 c와 n₀**가 존재하면


f(n) = O(g(n))


이라고 표현합니다.

---

# 🔍 빅오 표기법의 의미

빅오 표기법은 알고리즘 실행시간을 **대략적으로 표현하기 위한 방법**입니다.

불필요한 정보는 제거하고 **가장 큰 영향을 주는 항만 남깁니다.**

---

# 📌 빅오 계산 규칙

### 1️⃣ 가장 높은 차수 선택

예


f(n) = 5n² + 2n + 7


가장 큰 차수


n²


---

### 2️⃣ 상수 제거


5n² → n²


---

### 3️⃣ 낮은 차수 제거


n² + n + 1 → n²


---

### 최종 결과


f(n) = O(n²)


---

# 📊 빅오 계산 예제

### Example 1


f(n) = 2n² + 3n + 1

O(n²)


---

### Example 2


f(n) = 3n³ + 5n² + 2n + 10

O(n³)


---

### Example 3


f(n) = n² + 100n + 10000

O(n²)


---

# 📉 시간 복잡도 증가 순서

알고리즘 복잡도는 다음 순서로 증가합니다.


O(1) < O(log n) < O(n) < O(n log n)
< O(n²) < O(n³) < O(2ⁿ)


---

# 💻 코드 예시

## O(n)

```c
for(int i = 0; i < n; i++)
{
    printf("%d", i);
}

반복문이 n번 실행

O(n)
O(n²)
for(int i = 0; i < n; i++)
{
    for(int j = 0; j < n; j++)
    {
        printf("%d", i + j);
    }
}

반복문이 n × n 실행

O(n²)
